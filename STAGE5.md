# 阶段 5：用户问答方式学习实现说明

## ✅ 已实现的功能

### 1. 独立的 UserPatternService

**实现位置：** `UserPatternService`

**核心功能：**
- 高频术语统计
- 触发词识别
- 问题模板提取
- 上下文记忆
- 自动清理（30天）

### 2. 轻量级学习策略

**设计原则：**
- 默认本地规则优先
- 只在必要时使用 LLM
- 不影响默认提问速度

**触发条件：**
- 命中触发词（词频 > 5）
- 检测到追问（包含代词、指代词）
- 其他情况：跳过高级分析

### 3. 高频术语统计

**工作方式：**
```
用户提问
  ↓
提取关键词（去除停用词）
  ↓
更新词频统计
  ↓
识别触发词（词频 > 5）
  ↓
保存到 user-patterns.json
```

**停用词过滤：**
- 中文：的、了、是、在、我、有、和...
- 英文：the, is, at, which, on, a, an...

### 4. 触发词选择性加载

**触发词定义：**
- 词频 > 5 的术语
- 最多保留前 50 个
- 按词频降序排序

**优化效果：**
```
普通问题（无触发词）：
  Query → 直接检索（快速）

命中触发词：
  Query → Standalone Question → Query Variants → 检索（完整分析）

追问：
  Query → Standalone Question → 检索（改写追问）
```

### 5. 问题模板识别

**识别模式：**
- "什么是" / "what is"
- "如何" / "how to"
- "为什么" / "why"
- "在哪" / "where"
- "谁" / "who"
- "有没有" / "is there"

**用途：**
- 了解用户常用问法
- 未来可用于问题推荐

### 6. 上下文记忆

**功能：**
- 记录最近 10 个上下文
- 用于追问改写
- 自动清理旧数据

**示例：**
```
用户："我的 Python 配置在哪？"
系统：记录上下文 {lastTopic: ["Python", "配置"]}

用户："它的版本是多少？"
系统：改写为 "Python 配置的版本是多少？"
```

## 🔄 集成到检索链路

### QueryAnalysisService 优化

**原流程：**
```
Query → Standalone Question → Query Variants → Filters
```

**优化后：**
```
Query
  ↓
检测触发词
  ↓
无触发词 → 直接检索（快速路径）
  ↓
有触发词/追问 → 完整分析（高级路径）
  ↓
Standalone Question → Query Variants → Filters
```

### 性能对比

| 场景 | 原方案 | 优化后 |
|------|--------|--------|
| 普通问题 | 3次 LLM 调用 | 0次 LLM 调用 |
| 命中触发词 | 3次 LLM 调用 | 3次 LLM 调用 |
| 追问 | 3次 LLM 调用 | 1次 LLM 调用 |

## 🎯 使用场景

### 1. 高频术语优化
- 用户经常问"Python"相关问题
- "Python"成为触发词
- 下次问"Python"时，使用完整分析

### 2. 追问改写
- 用户："我的配置文件在哪？"
- 用户："它的格式是什么？"
- 系统改写："配置文件的格式是什么？"

### 3. 快速路径
- 用户："今天天气怎么样？"（无触发词）
- 系统：跳过高级分析，直接检索
- 速度更快
